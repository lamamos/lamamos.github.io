Get involved

You don't need to know how to code to help the project. The first thing you can do is just to talk to people. Not necessarily about this project, but about the issues we are facing (cf the why page.). One needs to see a problem before starting to solve it.

Then if you want to contribute more directly to the project you can report us any strange things in the software. That means any bugs (the soft doesn't do what I want it to do), but also features you'd like to see added to the software.

The goal of Lamamos is to provide tools usable by anyone. That means that everything must be translated into every imaginable languages. If you'd like to translate a part of the software in an language that is not yet supported, you can add it to that filea>. If you are not comfortable with git and these coding tools just send us a mail we can simplify the task for you.

You might have guessed that the design, logo and visuals of the site and everything until now have been done by a developer. You think you can do better, just do it and send us the result by mail. If you have any questions just ask us by mail also. ;)

An other way to make the project accessible is to write documentation addapted to every public. Don't hesitate to write your own tutorials, and to contribute to the documentations of the project which is here.

Finaly just fork a project and start toying around with it. Break it, repear it, improve it and send us a pull request. The project page is here.
